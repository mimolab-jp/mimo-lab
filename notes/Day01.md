# VS Code

## 学んだこと

Visual Studio Code（VS Code）は現在最も利用されているコードエディタの一つ。

メモ帳とは違い、拡張機能を追加して自分だけの開発環境を作ることができる。

---

VS Codeに後から機能を追加する仕組み。

スマホでいう「アプリ」のようなもの。

今日導入したもの

* Live Server
* Japanese Language Pack
* Auto Rename Tag

---

# Live Server

## 学んだこと

保存（Ctrl + S）するとブラウザが自動更新される。

昔

```text
Ctrl + S

↓

Alt + Tab

↓

F5
```

今

```text
Ctrl + S

↓

終わり
```

ブラウザ更新を意識しなくてよくなった。

---

# localhost

Live Serverが自分のPCの中だけに作るWebサーバー。

他の人からは見えない。

安心して実験できる。

---

# カラーピッカー

CSSの色指定では

```css
background: #fff3f7;
```

などのカラーコードの前に表示される□をクリックするとカラーピッカーが開く。

昔のようにカラーコード表を探してコピペする必要がなくなった。

---

# Auto Rename Tag

開始タグを書き換えると閉じタグも自動で変更される。

例

```html
<div>
```

↓

```html
<section>
```

すると

```html
</div>
```

↓

```html
</section>
```

に自動で変更される。

---

# シンタックスハイライト

コードの種類ごとに色分けして表示する機能。

例

* HTMLタグ
* CSSプロパティ
* 属性
* 文字列
* 色コード

昔のメモ帳では全部黒文字だったため、タグ探しが大変だった。

# class

HTML

```html
<div class="card">
```

CSS

```css
.card
```

この2つが対応している。

classは「名前」だと考えると理解しやすい。

---

# display

よく使う値

```css
display: none;
```

表示しない。

```css
display: block;
```

通常表示。

```css
display: flex;
```

レイアウト用。

今日学んだこと

JavaScriptからdisplayを書き換えるとCSSよりJavaScriptが優先される。

# 今日の気付き

コードを書くこと以上に、

**「使いやすいか？」**

を考えることが重要。

実際に、

* ホーム画面の下にラーメンページが表示されてしまった
* 戻るボタンが大きすぎた

という問題を修正した。

「動けばOK」ではなく、「気持ちよく使えるか」を考えることがWeb制作では大切。
