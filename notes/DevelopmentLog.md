# Day03 午前（2026-07-02）

## 今日のテーマ

ラーメンカードに写真を表示する。

## 実装したこと

### imagesフォルダ作成

画像保存用に `images` フォルダを作成。
画像ファイル名は半角英数字で管理する方針にした。

### データへ画像項目を追加

ラーメンデータに `image` を追加。

```javascript
image: "images/kiboshi.png"
```

画像もラーメン1件分のデータとして管理できるようになった。

### カードに画像を表示

`renderRamenList()` の中で、画像がある場合だけ表示する処理を追加。

```javascript
${ramen.image ? `
  <img class="ramen-photo" src="${ramen.image}" alt="${ramen.name}">
` : ""}
```

画像がないカードでは何も表示しない。

### 編集フォームに画像パス欄を追加

LocalStorageに保存済みの既存データにも画像を追加できるよう、編集フォームに画像パス欄を追加。
保存時に `image` も保存するようにした。

### 写真表示CSS

```css
.ramen-photo {
  width: 100%;
  height: 180px;
  object-fit: cover;
  border-radius: 16px;
  margin-bottom: 16px;
}
```

一覧カードでは写真をサムネイル風に表示。
`object-fit: cover;` により、枠いっぱいに画像を表示し、はみ出す部分はトリミングされる。

## 今日理解したこと

### function と実行命令の違い

```javascript
function renderRamenList() {
}
```

これは機能の定義。

```javascript
renderRamenList();
```

これはその機能を実行する命令。
「作っておく」と「今動かす」は別。

### LocalStorageと初期データ

LocalStorageに保存済みデータがある場合、`app.js` の初期データよりLocalStorageが優先される。
そのため、初期データに `image` を追加しても、保存済みデータ側に画像項目がなければ表示されない。

### デバッグ

写真が出ない原因を、Console・LocalStorage・画像パス・JavaScript文法の順に確認した。
カンマ抜け、画像パス違い、保存済みデータの影響を実体験した。

### object-fit

`cover` は枠を埋める。
`contain` は画像全体を枠内に収める。
一覧カードでは `cover` の方が写真の印象が強くなり、ラーメンの記憶を呼び起こすサムネイルになる。

### Responsive Design

正式名称は Responsive Design。
業界では「レスポンシブ」と略される。
画面サイズに反応して、レイアウトや幅を変えるデザインのこと。

## 今日のみもメモ

* 写真が出るまで長かった。
* でもその過程でデバッグを体験できた。
* `renderRamenList()` の定義と実行の違いが分かった。
* `width: 100%;` に違和感を持ったが、`object-fit: cover;` と組み合わせることで美味しそうなドアップ表示になると理解した。
* 写真が入ると一気にアプリ感が増した。
* 「全部見せる」より「記憶を呼び起こす」写真配置の良さを感じた。
